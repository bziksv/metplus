# Визуальный язык каталога (product-hint)

Единая система бейджей, заблокированных полей, легенды и тултипов в таблице товаров.

**Где смотреть локально:**

- http://localhost:8086/catalog/stal_armaturnaya_a3/ — арматура (бейдж **500+**, динамическая блокировка полей)
- http://localhost:8086/catalog/list_g_k/ — лист (бейдж **замок**, фиксированные метры/ширина)

---

## Общие принципы

| Элемент | Правило |
|---------|---------|
| Плашки | `border-radius: 4px`, светлый фон, тонкая рамка `1px` |
| Замок | Один SVG-значок везде: бейдж, ячейка, легенда |
| **500+** | Голубая плашка — отдельно от серой «замковой» |
| Легенда | Цветные маркеры: зелёный / жёлтый; для особых условий — тот же бейдж, что у товара (**500+**, замок) |
| Тултипы | Тёмный фон `#2f4052`, белый текст; показ по hover, focus и клику |

## CSS-классы

### Бейджи (рядом с названием товара)

| Класс | Назначение | Вид |
|-------|------------|-----|
| `.product-hint` | Базовая плашка | Серая, `cursor: help` |
| `.product-hint--lock` | Только целыми штуками (`TOLKO_SHT`) | Серая + иконка замка |
| `.product-hint--bulk` | Заказ от N кг (`SHT_M_VES_OT500_KG` / `SHT_M_VES_OT_1000_KG`) | Голубая + текст `500+` / `1000+` |
| `.product-hint__icon--lock` | Иконка замка | SVG из `--hint-lock-icon` |
| `.product-hint__label` | Текст внутри бейджа (`500+`) | Мелкий жирный шрифт |

### Заблокированные ячейки таблицы

| Класс | Назначение |
|-------|------------|
| `.product-table_field--restricted` | Поле только для чтения (замок + значение или input readonly) |
| `.product-table_field--synced` | Вес считается из штук (до 500 кг), серый фон, но редактируемый при фокусе |
| `.product-table_cell--locked` | Ячейка с заблокированным вводом (метры/штуки в bulk-режиме) |

Динамическая блокировка на арматуре **500+** (в режиме заказа по весу ≥ 500 кг) добавляет замок в метры/штуки через `setFieldRestricted()` в `main.js` — тот же `.product-hint__icon--lock`.

### Легенда под таблицей

**Порядок (всегда сверху вниз):**

1. `.product-hint-legend--stock` — зелёный квадрат — «Наличие товара на складе»
2. `.product-hint-legend--limited` — жёлтый квадрат — «Количество ограничено…»
3. **Доп. условия** — только если в разделе есть такие товары (см. правило ниже)

| Класс | Маркер | Условие показа | Текст |
|-------|--------|----------------|-------|
| `.product-hint-legend--bulk` | Бейдж **500+** / **1000+** (как у товара) | `BULK_WEIGHT_THRESHOLDS` | От N кг можно заказать по весу |
| `.product-hint-legend--lock` | Бейдж-замок (как у товара) | `HAS_ONLY_PIECES_ROWS` | Только целыми штуками |
| `.product-hint-legend--neutral` | Серый квадрат | редактируемый вес, но нет 500+ | Подсказка по пересчёту веса |

**Правило доп. подсказок:** если в категории есть товары с особыми свойствами (`TOLKO_SHT`, `SHT_M_VES_OT500_KG`, пересчёт веса) — под базовыми строками легенды выводим пояснение с **тем же бейджем**, что в строке товара, и кратким текстом. Полное описание — в `data-tip` (тултип по hover/клику).

Разметка строки с бейджем:

```html
<div class="product-hint-legend product-hint-legend--badge product-hint-legend--bulk" data-tip="…">
    <span class="product-hint-legend__badge product-hint product-hint--bulk" aria-hidden="true">
        <span class="product-hint__label">500+</span>
    </span>
    <span class="product-hint-legend__text">От 500 кг можно заказать по весу</span>
</div>
```

### Тултипы

Атрибут `data-tip` на любом элементе с классом из системы. Стили в `[data-tip]::before` (`custom.css`).

- Показ: `:hover`, `:focus-visible`, класс `.is-tip-visible` (клик — в `main.js`)
- Для бейджей и заблокированных полей — смещение влево (не по центру)
- **Исключение:** `.product-hint-legend[data-tip]` — маркер в `::before`, тултип в `::after` (чтобы не затирать цветной квадрат/замок)

## Иконка замка

Один SVG (скобка + корпус), задаётся CSS-переменной:

```css
--hint-lock-icon: url("data:image/svg+xml,...");
```

Используется в:

1. `.product-hint__icon--lock` — бейдж и ячейка
2. `.product-hint-legend--lock::before` — маркер в легенде (на серой плашке 4px)

Не рисовать замок через `::before`/`::after` — только этот SVG.

## Бизнес-логика → UI

| Свойство товара | Бейдж | Поведение полей |
|-----------------|-------|-----------------|
| `TOLKO_SHT = Да` | Замок | Метры и ширина заблокированы, штуки — целые |
| `SHT_M_VES_OT500_KG = Да` | 500+ | До 500 кг — штуки; от 500 кг — вес, метры/штуки блокируются |
| `SHT_M_VES_OT_1000_KG = Да` | 1000+ | До 1000 кг — штуки; от 1000 кг — вес, метры/штуки блокируются |
| Обычная арматура с весом | — | Вес / метры / штуки синхронизируются свободно |

## Проверка после правок

1. **stal_armaturnaya_a3** — ф12: бейдж `500+` голубой; в легенде после жёлтого квадрата — бейдж `500+` и текст «От 500 кг…»
2. **list_g_k** — бейдж-замок у товаров `TOLKO_SHT`; в легенде — бейдж-замок и «Только целыми штуками»
3. Замок в бейдже, ячейке и легенде визуально одинаковый
4. Тултип тёмный при наведении и по клику на бейдж/поле с `data-tip`
